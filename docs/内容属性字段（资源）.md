# MCP 内容属性字段模型

1. ## MCP 字段模型
    

2. ### 基本信息（强结构：5字段）
    

- 标签（树形类目）：`category_ids[]`
    
- 名称：`name`
    
- 提供方：`provider_name`
    
- 仓库地址：`repo_url`
    
- 用途一句话：`one_liner`
    

2. ### 案例展示（强结构：3字段 + 可变媒体区；支持多组）
    

- `cases[]`（至少 1 组）
    
    - 用户输入：`user_input`
        
    - 执行过程：`execution_process`
        
    - Agent输出：`agent_output`
        
    - 展示区（可变）：`media[]`（0..N，图/视频/混合）
        

3. ### 如何使用（弱结构：3段；字段内原样存“JSON文本/块”）
    

- D1 Json配置（必填）：`how_to_use.json_config_text`
    
- D2 常用客户端（可空）：`how_to_use.common_clients_json`
    
- D3 运行形态补充（可空）：`how_to_use.runtime_modes_json`
    

> 说明：这三段 **不做结构约束**，只保证“能存、能原样展示/复制”。

### 1.1.4 讨论区

- 复用 `comment(target_type='mcp', target_id=mcp_id)`
    

---

2. ## MCP 记录字段（系统必备，不属于内容块但必须落库）
    

- `mcp_id`（主键）
    
- `source`：`official|user`
    
- `author_user_id`
    
- `publish_status`：`draft|pending|published|rejected|unpublished`
    
- `is_verified`（后台人工维护）
    
- `created_at` `updated_at`
    

---

3. ## MCP 数据字典（字段路径 → 类型/必填/多值/说明）
    

|   |   |   |   |   |
|---|---|---|---|---|
|字段路径|类型|必填|多值|说明|
|basic.category_ids|array|否|Y|树形类目ID列表（逻辑字段；DB可用绑定表）|
|basic.name|string|是|N|MCP名称|
|basic.provider_name|string|是|N|提供方/组织名|
|basic.repo_url|string(url)|是|N|仓库地址|
|basic.one_liner|string|是|N|用途一句话|

|cases[].execution_process|text|是|N|执行过程|
|cases[].agent_output|text|是|N|Agent输出|
|cases[].media|array|否|Y|可变展示区（图/视频等）|

|cases[].media[].asset_id|string|否|N|关联站内文件资产ID（有则填）|

4. ## MCP 表结构总览
    

- `mcp`：系统字段 + 基础信息（5字段）
    
- `mcp_howto`：如何使用（D1/D2/D3，原样存）
    
- `mcp_case`：案例展示（多组）
    
- `mcp_case_media`：案例展示区媒体（多图/多视频）
    
- 类目绑定：复用你已有 `category_binding`（`target_type='mcp'`）
    

---

### 1) `mcp`（主表：系统 + 基础信息）

**用途**：承载“强结构 5 字段” + 生命周期字段。

|   |   |   |   |
|---|---|---|---|
|字段|类型|必填|说明|
|mcp_id|bigint PK|Y|主键|
|source|varchar(16)|Y|`smithery|
|author_user_id|bigint|Y|提交者|
|publish_status|varchar(16)|Y|`draft|
|is_verified|bool|Y|后台人工维护|
|location|varchar(8)|Y|`local|
|name|varchar(200)|Y|基础信息：名称|
|provider_name|varchar(200)|Y|基础信息：提供方|
|repo_url|varchar(500)|Y|基础信息：仓库地址|
|one_liner|varchar(300)|Y|基础信息：用途一句话|
|created_at|datetime|Y||
|updated_at|datetime|Y||

**索引/约束**

- INDEX `idx_mcp_list (publish_status, is_verified, location, source)`
    
- INDEX `idx_mcp_author (author_user_id)`
    

> 类目：用 `category_binding(target_type='mcp', target_id=mcp_id, category_id)`，允许多类目/可为空。

---

### 2) `mcp_howto`（1:1：如何使用，弱结构）

**用途**：D1 必填；D2/D3 可空；全部原样存 text/json。

|   |   |   |   |
|---|---|---|---|
|字段|类型|必填|说明|
|mcp_id|bigint PK/FK|Y|1:1|
|json_config_text|longtext|Y|D1：JSON配置原文（可含多段/注释）|
|common_clients_json|longtext|N|D2：常见客户端（原样存）|
|runtime_modes_json|longtext|N|D3：运行形态补充（原样存）|
|created_at|datetime|Y||
|updated_at|datetime|Y||

**约束**

- FK：`mcp_howto.mcp_id -> mcp.mcp_id`（cascade delete）
    

---

### 3) `mcp_case`（多组案例）

**用途**：每个 MCP ≥1 组（DB不强制，发布上架前校验）。

|   |   |   |   |
|---|---|---|---|
|字段|类型|必填|说明|
|case_id|bigint PK|Y||
|mcp_id|bigint FK|Y|归属MCP|
|order_no|int|Y|展示顺序|
|user_input|longtext|Y|案例：用户输入|
|execution_process|longtext|Y|案例：执行过程|
|agent_output|longtext|Y|案例：Agent输出|
|created_at|datetime|Y||
|updated_at|datetime|Y||

**索引/约束**

- INDEX `idx_case_mcp (mcp_id, order_no)`
    

---

### 4) `mcp_case_media`（案例媒体：可变展示区）

**用途**：支持多图/多视频/混合；不解析媒体内容。

|   |   |   |   |
|---|---|---|---|
|字段|类型|必填|说明|
|media_id|bigint PK|Y||
|case_id|bigint FK|Y|归属案例|
|order_no|int|Y|顺序|
|media_type|varchar(16)|Y|`image|
|asset_id|bigint|N|站内文件（FK file_asset 可选）|
|external_url|varchar(800)|N|外链（可选）|
|caption|varchar(300)|N|说明|
|created_at|datetime|Y||
|updated_at|datetime|Y||

**约束**

- INDEX `idx_media_case (case_id, order_no)`
    
- 业务校验：`asset_id` 与 `external_url` **至少填一个**
    

---

5. ## MCP 发布校验规则（必须写进服务端）
    

- `mcp_howto.json_config_text` 必填
    
- `mcp_case` 至少 1 条
    
- `publish_status=pending/published` 前做上述校验
    
- “zip 风险提示”不进结构化字段：下载时统一提示（全局文案或前端固定文案）

# Skill 内容属性字段模型

## 2.1 Skill 字段模型

### 2.1.1 基本信息（强结构）

- 标签：用**树形类目**（必须 ≥1 个）
    
- 名称 / 作者 / 仓库 / Stars / Forks / 最近更新 / 用途一句话 （无 GitHub 链接时：仓库&Stars/Forks/最近更新为空；作者自动填“上传者昵称”）
    

### 2.1.2 案例展示（强结构：3字段 + 可变媒体区；多组）

- `cases[]`（每个 Skill **至少 1 组**，支持多组）
    
    - `user_input`
        
    - `execution_process`
        
    - `agent_output`
        
    - `media[]`（0..N，image/video 混合）
        

### 2.1.3 如何使用（强结构）

- 安装命令：结构化数组（npx/bunx/pnpm dlx…）
    
    - **有 GitHub 链接**：由系统根据 repo 拼出命令并存
        
    - **无 GitHub 链接**：为空
        
- zip 下载：**每个 Skill 必须有 zip**（统一走 `file_asset`）
    
- 使用说明：**无 GitHub 链接时必须有**（有 GitHub 时可选）
    

### 2.1.4 讨论区

- 复用 `comment(target_type='skill', target_id=skill_id)`
    

---

- `author_user_id`
    
- `publish_status`：`draft|pending|published|rejected|unpublished`

- `created_at` `updated_at`

# Prompt 内容属性字段模型

---

## 3.1 Prompt 字段模型

### 3.1.1 效果展示（可变展示区）

- `showcase_items[]`：支持图片/视频/纯文本介绍；可混合、多条
    

### 3.1.2 基本信息（强结构）

- 类目标签（树形类目，必有，支持多类目）
    
- 模型/平台（如 Midjourney）
    
- 作者（user）
    
- 语言 `language`
    

### 3.1.3 Prompt 本体（核心）

- `prompt_text`（长文本，原样存）
    

### 3.1.4 评论区

- 复用：`comment(target_type='prompt', target_id=prompt_id)`
    

---

## 3.2 Prompt 记录字段 （系统必备）

- `prompt_id`（主键）
    
- `source`：`official|user`
    
- `author_user_id`
    
- `publish_status`：`draft|pending|published|rejected|unpublished`
    
- `is_verified`（后台人工维护）
    
- `created_at` `updated_at`
    

  

---

## 3.3 Prompt 数据字典（逻辑字段路径）
