# API 契约（REST / Cookie Session / PG FTS）

> 产出：**API清单 + Schema + 路径字典**
> 
> 统一：**按类型资源路由**（`/prompts /mcps /skills /tutorials`），**状态机动作**走 `POST /contents/{id}:action`
> 
> ID：Snowflake BIGINT —— **API 里一律用 string 返回/传入**（避免 JS 精度问题）

---

1. ## 约定（必须遵守）
    

### 0.1 鉴权

- 会话：**Cookie**（有状态 session），默认 3 天，滑动续期
    
- 同一用户最多 **3 个 device_id 在线**
    
- 暂不需要 CSRF
    

### 0.2 可见性（状态）

- `Draft / PendingReview / Reject / Approved / Unlisted`：**作者 + 审核员 + 管理员**可见；其他人访问返回 **403**
    
- `Listed`：所有人可见（登录可选）
    
- 权限失败：**403 +** **`ERR_PERMISSION`**（即使是“看不到”的资源也用 403，不用 404）
    

### 0.3 通用响应包

**Success**

```JSON
{ "ok": true, "data": {}, "meta": {} }
```

**Error**

```JSON
{
  "ok": false,
  "error": {
    "code": "ERR_PERMISSION",
    "message": "permission denied",
    "details": {}
  }
}
```

### 0.4 通用分页（offset/limit）

- 入参：`offset`（默认 0）、`limit`（默认 20，最大 100）
    
- 出参：
    

```JSON
"meta": { "offset": 0, "limit": 20, "total": 123 }
```

### 0.5 通用排序

- 入参：`sort` ∈ `hot_score | created_at | views_7d`
    
- 入参：`order` ∈ `asc | desc`（默认 `desc`）
    
- 默认：`sort=hot_score&order=desc`
    
- `hot_score` 口径：`views_7d*1 + up_7d*5 + comments_7d*3`
    

### 0.6 通用搜索（PG FTS）

- 入参：`q`（全文检索关键字）
    
- 入参：`highlight`（`0|1`，默认 0）
    
- 支持组合：`q` + `type/category/tag` 同时使用
    
- 高亮返回：`highlight` 字段（见 Schema）
    

### 0.7 文件上传（直传两段式）

- `POST /assets/presign`：拿到 presigned url + 临时凭证
    
- 前端直传对象存储
    
- `POST /assets/commit`：提交落库确认
    
- 限制：最大 500MB；允许 mime：`pdf/doc/docx/md/txt/png/jpg/jpeg/zip`

2. ## Schema（核心对象）
    

> 仅列字段口径（类型+必填）；时间一律 ISO8601 字符串；ID 一律 string。

### 2.1 ContentSummary（列表卡片）

```JSON
{
  "id": "string",
  "type": "prompt|mcp|skill|tutorial",
  "status": "Draft|PendingReview|Reject|Approved|Listed|Unlisted",
  "title": "string",
  "one_liner": "string|null",
  "category": { "id":"string", "name":"string" },
  "tags": [{"id":"string","name":"string"}],
  "author": { "id":"string", "nickname":"string", "avatar_asset_id":"string|null" },
  "cover_asset_id": "string|null",
  "stats_7d": { "views":"int", "up":"int", "comments":"int", "hot_score":"int" },
  "created_at": "iso",
  "updated_at": "iso",
  "highlight": {
    "title": "string|null",
    "one_liner": "string|null",
    "document": "string|null"
  }
}
```

### 2.2 ContentBase（详情公共部分）

```JSON
{
  "id":"string","type":"string","status":"string",
  "title":"string","one_liner":"string|null",
  "category_id":"string","tag_ids":["string"],
  "author_id":"string","cover_asset_id":"string|null",
  "created_at":"iso","updated_at":"iso"

```

### 2.3 Status 枚举（动作依赖）

- `Draft / PendingReview / Reject / Approved / Listed / Unlisted / Deleted`
    
- 说明：`Deleted` 为“删除请求/日志语义”，实际为物理删除

3. ## API 清单（按模块）
    

> 说明：**列表**返回 `ContentSummary[]`；**详情**按 type 拆分；**动作**统一 `POST /contents/{id}:action`。

### 3.1 Auth / Session

- `POST /auth/sms:send` 发送验证码（可假发送）
    
- `POST /auth/login` 手机号+验证码登录（Set-Cookie）
    
- `POST /auth/logout` 退出（撤销 session）
    
- `GET /me` 当前用户信息
    
- `GET /me/sessions` 当前在线设备/会话
    
- `DELETE /me/sessions/{session_id}` 踢下线
    

### 3.2 内容：Prompt

- `GET /prompts` 列表（支持：分页/筛选/搜索/高亮/排序）
    
- `POST /prompts` 创建（默认 `Draft`）
    
- `GET /prompts/{id}` 详情（PromptDetail）
    
- `PATCH /prompts/{id}` 编辑（保存后可能自动转 `PendingReview`，见 4.1）
    

### 3.3 内容：MCP

- `GET /mcps`
    
- `POST /mcps`
    
- `GET /mcps/{id}`（McpDetail）
    
- `PATCH /mcps/{id}`
    

### 3.4 内容：Skill

- `GET /skills`
    
- `POST /skills`
    
- `GET /skills/{id}`（SkillDetail）
    
- `PATCH /skills/{id}`
    
- `POST /skills/{id}:download` 下载（若 repo_url 有且 zip 缺失：触发首次生成并回写 zip_asset_id）
    

### 3.5 内容：Tutorial

- `GET /tutorials`
    
- `POST /tutorials`
    
- `GET /tutorials/{id}`（TutorialDetail）
    
- `PATCH /tutorials/{id}`
    

### 3.6 内容状态机动作（统一）

- `POST /contents/{id}:submit_review`
    
- `POST /contents/{id}:approve`（审核员/管理员）
    
- `POST /contents/{id}:reject`（审核员/管理员）
    
- `POST /contents/{id}:list`（上架：Approved→Listed）
    
- `POST /contents/{id}:unlist`（下架：Listed→Unlisted）
    
- `POST /contents/{id}:delete`（作者可删自己的；物理删除；级联清理互动/统计/事件）
    

### 3.7 评论（一次性树；顶层/子评论均 desc）

- `GET /contents/{id}/comments` 返回 CommentNode 树
    
- `POST /contents/{id}/comments` 发评论（body：`content_text`,`parent_id?`）
    
- `DELETE /comments/{comment_id}` 删除自己的评论（置 deleted + 清空 content_text）
    

### 3.8 投票（PUT/DELETE 幂等；返回当前态）

- `PUT /contents/{id}/vote` body：`{"vote":"up|down"}`
    
- `DELETE /contents/{id}/vote`
    

### 3.9 收藏/资产库

- `GET /me/favorite-folders`（树，≤5 层）
    
- `POST /me/favorite-folders` 创建文件夹
    
- `PATCH /me/favorite-folders/{id}` 重命名/移动
    
- `DELETE /me/favorite-folders/{id}` 删除（实现可选择：迁移到根目录/阻止删除非空）
    
- `PUT /contents/{id}/favorite` body：`{"folder_id":"string"}`
    
- `DELETE /contents/{id}/favorite`
    

### 3.10 榜单（统一接口 + type 参数）

- `GET /ranks` query：`type=prompt|mcp|skill|tutorial&window_days=7&date=YYYY-MM-DD`
    
    - 返回：`[{rank, score, content: ContentSummary}]`
        

### 3.11 附件（直传）

- `POST /assets/presign` 申请直传
    
- `POST /assets/commit` 提交确认
    
- `GET /assets/{id}` 获取资产信息（含 url）
    

**presign 请求**

```JSON
{ "filename":"a.zip", "mime_type":"application/zip", "size_bytes": 123 }
```

**presign 响应**

```JSON
{
  "asset_temp_id":"string",
  "upload": { "url":"string", "method":"PUT", "headers":{} },
  "expires_at":"iso"
}
```

**commit 请求**

```JSON
{ "asset_temp_id":"string", "storage_key":"string", "etag":"string|null" }
```

**commit 响应**

```JSON
{ "asset": { "$ref":"FileAsset" } }
```

4. ## 行为规则（与 API 对齐）
    

### 4.1 编辑导致状态变化

- 允许编辑状态：`PendingReview / Listed / Reject`
    
- **保存后自动转** **`PendingReview`**
    

### 4.2 重新上架

- `Unlisted` 未修改保存：可直接转 `Approved`（再 list）
    
- 一旦保存修改：转 `PendingReview`
    

### 4.3 最小结构校验（submit_review / list 前强校验）

- 所有类型：`title` 必填；`category_id` 必填
    
- Prompt：`model_name/language/prompt_text` + `showcases>=1`
    
- MCP：`source/provider/repo_url` + `cases>=1` + `howto.standard_config/clients`
    
- Skill：`cases>=1`；`repo_url` 为空则 `zip_asset_id` 必填
    
- Tutorial：`body_markdown` + `media`（按产品规则）

5. ## 后台域（Admin）
    

> 路径前缀：`/admin/...`（仅审核员/管理员可访问）

### 5.1 分类/标签 CRUD

- `GET /admin/categories` 列表（树）
    
- `POST /admin/categories`
    
- `PATCH /admin/categories/{id}`
    
- `DELETE /admin/categories/{id}`
    
- `GET /admin/tags`
    
- `POST /admin/tags`
    
- `PATCH /admin/tags/{id}`
    
- `DELETE /admin/tags/{id}`
    

### 5.2 审核 Case（列表/查看）

- `GET /admin/moderation/cases` query：`status=pending|approved|rejected&type=...&q=...`
    
- `GET /admin/moderation/cases/{case_id}`
    

> 审核动作仍走：`POST /contents/{id}:approve` / `:reject`

### 5.3 事件日志查询（不含 PV）

- `GET /admin/events` query：`type=event_type&content_id&actor_user_id&date_from&date_to&offset&limit`
    

### 5.4 审计日志查询

- `GET /admin/audit-logs` query：`action_type&target_type&target_id&actor_user_id&date_from&date_to&offset&limit`
